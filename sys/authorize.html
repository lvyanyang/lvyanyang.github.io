# 权限认证

接口调用可以设置权限，权限最小粒度为请求方法。

# 使用方法

  1. 如果要求接口调用者必须登录之后(接口调用时HttpHeader中添加有效的token值)才能访问，需要在所在ApiController(类级生效)或者ApiController方法(方法级生效)上使用@Authorize注解。
  2. 如果要指定的权限，需要在@Authorize注解上添加code属性，@Authorize(code = "sys.dept.select")，code值是在模块中定义的编码，多个权限编码之间用逗号隔开,如有多个编码需要同时满足才能通过认证。
  3. 接口Controller和页面Controller用法一致。

# 实现原理

权限认证是通过拦截器实现的，拦截的地址是在xci.api.intercept-url-patterns的配置值，拦截器定义在SysApplicationModule模块中。

权限编码实现权限校验的完整流程：

  1. 添加模块。
  2. 添加角色，并给角色分配可访问的模块。
  3. 添加用户，并分配一个或者多个角色，多个角色的模块会合并到用户中。
  4. 在需要设置权限的ApiController或者方法上设置@Authorize(code = "权限编码1,权限编码2...")注解。
