# 机构数据权限

在项目开发过程中经常有这样一种需求，需要根据不同的人显示不同机构的数据。本质上就是根据配置，在后台生成不同的过滤SQL实现结构数据权限。

目前支持的数据权限分为以下几种：

  1. 全部
  2. 自定义
  3. 所在机构及所有下级
  4. 所在机构
  5. 仅本人

由于每个角色都可以设置机构数据权限，当一个用户拥有多个角色时，多个角色中的机构数据权限会进行合并，合并策略在系统参数sys.dept.scopeMergeStrategy进行配置。

机构数据权限合并策略：

  1. 有更大的权限时优先采用更大的权限
  2. 有更小的权限时优先采用更小的权限

# 使用方法

  1. 在需要使用机构数据权限的服务方法设置@DeptScope注解。
@DeptScope
public List<SysRole> selectList(SysRoleFilter filter);

  1. 方法参数中，必须要有一个实现了IDeptScope接口的参数
public class SysRoleFilter extends BasePageFilter implements IDeptScope;

  1. 在MybatisXML中的SQL中，添加与机构表的关联。
select r.id,r.name,d.id dept_id,d.name dept_name
from sys_role r left join sys_dept d on r.dept_id = d.id

# 注意事项

  1. 用启用机构数据权限，必须要使用@Authorize标注到控制器中，客户端Header中必须要传递token值，否则无法获取用户信息。
  2. @DeptScope注解可以设置属性deptIdName，表示关联机构表主键名称。默认值:d.id,d表示机构表别名。
  3. 方法参数中设置enableDeptScope来控制是否启用机构数据权限过滤。
  4. 超管用户直接返回所有数据。
  5. 如果账号停用，或者所属机构无效，则会抛出异常。
