# 权限模块

实现部门管理、角色用户、菜单及按钮授权、数据权限、系统参数、日志管理、通知公告等基础服务。

  • 权限模块的所有接口Url都以 /api 开头。

# 使用方法

项目文件 pom.xml 中添加依赖：

<!--XCI系统权限模块-->
<dependency>
    <groupId>com.github.lvyanyang</groupId>
    <artifactId>xci-framework-sys</artifactId>
    <version>${xci.framework.version}</version>
</dependency>
<!--XCI系统权限UI模块(UI可选)-->
<dependency>
    <groupId>com.github.lvyanyang</groupId>
    <artifactId>xci-framework-sys-ui</artifactId>
    <version>${xci.framework.version}</version>
</dependency>

# 接口请求Header公共参数

系统权限模块进行接口请求时要求传入以下功能请求参数，根据实际需要进行传递。

名称 说明
X-Requested-Type 接口请求标识，固定值：APIHttpRequest(必填)
appId 调用方标识 (选填)
timestamp 当前时间戳 (选填)
token 登录成功后返回的用户token(选填)

# AppId

appId表示应用标识，可以用来区分客户端身份或者类型，例如WinForm客户端使用appId=winApp，安卓客户端使用appId=androidApp，客户端进行请求时，服务端可以根据appId来确定是安卓客户端还是WinForm客户端，同时也可以判断客户端传递的appId是否正确(可选)，对于非法客户端的请求就可以直接拦截。

可通过参数配置，是否需要在接口调用时进行AppId验证。如果配置为不验证AppId，可以省略此参数

设置方式：

在项目配置文件application.properties中添加 xci.app.应用ID名称=应用ID值，可以添加多个。

#winform是appId，20201001是appId的值，客户端传值时，在HttpHeader中添加键值对：`appId=winform`。
xci.app.winform=20201001
xci.app.android=11870982334

# Timestamp

表示格林威治时间1970年01月01日00时00分00秒(北京时间1970年01月01日08时00分00秒)起至现在的总毫秒数。

可通过参数配置，是否需要在接口调用时进行时间戳验证，防接口无限重放。如果配置为不验证时间戳，可以省略此参数

Java中获取方法

System.currentTimeMillis();
1

Js中获取方法

new Date().getTime();
1

C#中获取方法

// 与 java 中的 System.currentTimeMillis() 值对应
(long)(DateTime.UtcNow - new DateTime(1970, 1, 1, 0, 0, 0, DateTimeKind.Utc)).TotalMilliseconds;
1
2

WARNING

时间戳本意是指格林威治时间1970年01月01日00时00分00秒(北京时间1970年01月01日08时00分00秒)起至现在的总秒数。

# Token

用户token主要用来表示用户身份，当用户在系统登录成功后，服务器会给每个用户分配一个独立无二的标识字符串，用来表示用户的身份，这样用户在后续的请求交互过程中就无需再传账号和密码来验证用户的身份了，只需要带上token就可以表示用户的身份了。

token是有时间期限的，过期之后就不能继续使用，需要重新登录，重新获取用户凭证，当前系统默认分配的token有效期是一个月，一个月后自动失效，开发者可以根据情况调整有效期。

如果接口不需要验证用户的身份，属于公开接口，此时也不需要传递此参数，只有控制器或者请求方法中设置了@Authorize注解，才会验证token参数。设置了@AllowAnonymous注解的控制器或者请求方法也不需要传递此参数。
