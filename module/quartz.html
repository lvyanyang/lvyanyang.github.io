# 定时任务模块

实现在线添加、修改、删除任务，控制任务启用、停用以及查看任务执行日志。 底层服务采用 quartz (opens new window) 框架。

# 介绍

首先介绍除了Quartz之外实现定时任务的几种简单方式：

  • Timer
  • ScheduledThreadPoolExecutor
  • @Scheduled

如需动态灵活的使用定时任务，请选择使用本模块。

# Timer

DETAILS 
public class TimerDemo {
    public static void main(String[] args) {
        java.util.Timer timer = new Timer();
        timer.schedule(new java.util.TimerTask() {
            @Override
            public void run() {
                System.out.println("TimerTask1 run" + LocalDateTime.now().format(DateTimeFormatter.ofPattern("HH:mm:ss")));
            }
        },1000,5000);//延时1s，之后每隔5s运行一次
    }
}

有两点问题需要注意：

  1. timer.schedule方法和timer.scheduleAtFixedRate方法的区别：scheduleAtFixedRate会尽量减少漏掉调度的情况，如果前一次执行时间过长，导致一个或几个任务漏掉了，那么会补回来，而schedule过去的不会补，直接加上间隔时间执行下一次任务。
  2. 同一个Timer下添加多个TimerTask，如果其中一个没有捕获抛出的异常，则全部任务都会终止运行。但是多个Timer是互不影响。

# ScheduledThreadPoolExecutor

DETAILS 
public class SchedulerDemo {
    public static void main(String[] args) {
        ScheduledExecutorService executorService = new ScheduledThreadPoolExecutor(5);
        
        //scheduleWithFixedDelay跟schedule类似，而scheduleAtFixedRate与scheduleAtFixedRate一样会尽量减少漏掉调度的情况。
        executorService.scheduleWithFixedDelay(new Runnable() {
            @Override
            public void run() {
                String now = LocalDateTime.now().format(DateTimeFormatter.ofPattern("HH:mm:ss"));
                System.out.println("ScheduledThreadPoolExecutor1 run:"+now);
            }
        },1,2,TimeUnit.SECONDS);
    }
}

# Scheduled

  1. 启动类添加@EnableScheduling
  2. 定时任务方法上添加@Scheduled
  3. 默认情况下@Scheduled是单线程允许，如需多线程，请自行设置ThreadPoolTaskScheduler。
DETAILS 
/**
 * 定时任务多线程配置，默认为单线程
 */
@Configuration
public class ScheduleConfig implements SchedulingConfigurer {
    @Override
    public void configureTasks(ScheduledTaskRegistrar scheduledTaskRegistrar) {
        ThreadPoolTaskScheduler taskScheduler = new ThreadPoolTaskScheduler();
        taskScheduler.setThreadNamePrefix("RunBusTask-");
        //设置线程池大小
        taskScheduler.setPoolSize(10);
        taskScheduler.initialize();

        scheduledTaskRegistrar.setTaskScheduler(taskScheduler);
    }
}


@Component
public class springScheduledDemo {
    //这里的cron表达式只支持6位，年份是不支持的，带年份的7位格式会报错：Cron expression must consist of 6 fields (found 7 in "1/5 * * * * ? 2018")
    @Scheduled(cron = "1/5 * * * * ?")
    public void testScheduled(){
        System.out.println("springScheduled run:" + LocalDateTime.now().format(DateTimeFormatter.ofPattern("HH:mm:ss")));
    }

任务表达式说明
  1. 定义一个组件，并编写组件方法

@Component("xciTask")
public class XciTask {
    public void runDefault() {
        log.info("默认任务");

    }
}

  1. 任务表达式直接写：xciTask.run('参数')来进行调用。

错误策略说明

在Quartz中，当一个持久的触发器因为调度器被关闭或者线程池中没有可用的线程而错过了激活时间时，就会发生激活失败(misfire)。

如果触发了misfire，处理策略如下：

名称 说明
立即执行 当服务恢复后错过的任务会立即执行
执行一次 当服务恢复后错过的任务会执行一次
放弃执行 当服务恢复后错过的任务将会被放弃

# 手动调用

QuartzService类实现了IJobService。如需手动调用请从Spring容器获取QuartzService实例直接调用。

查看代码 
@Slf4j
@Controller
@RequestMapping("/")
public class DefaultController extends BaseWebController {
    @Resource private QuartzService quartzService;
    
    public void taskDemo() {
        log.info("taskDemo...");
    }

    @ResponseBody
    @GetMapping(value = "/task")
    public Object task() {
        //调度一个任务，每2秒触发一次
        var job = quartzService.scheduleJob("defaultController.taskDemo()", "0/2 * * * * ?");
        // quartzService.pauseJob(job.getId());//暂停任务
        // quartzService.deleteJob(job.getId());//删除任务
        return BoolResult.success();
    }
}

# 调度任务服务接口

IJobService接口定义了定时调度任务的核心功能。

IJobService 
/**
 * 定时任务操作接口
 */
public interface IJobService {
    /**
     * 清除任务
     */
    void clearJob();

    /**
     * 恢复任务
     * @param jobId 任务ID
     */
    void resumeJob(Long jobId);

    /**
     * 暂停任务
     * @param jobId 任务ID
     */
    void pauseJob(Long jobId);

    /**
     * 删除任务
     * @param jobId 任务ID
     */
    void deleteJob(Long jobId);

    /**
     * 手动执行一次任务
     */
    BoolResult runOnceJob(IJobModel job);

    /**
     * 调度任务
     */
    void scheduleJob(IJobModel job);

    /**
     * 验证cron表达式的有效性
     * @param cronExpression cron表达式
     */
    BoolResult validCronExpression(String cronExpression);

    /**
     * 获取下一个执行时间根据给定的Cron表达式
     * @param cronExpression Cron表达式
     * @return Date 下次Cron表达式执行时间
     */
    Date getCronExpressionNextExecutionDate(String cronExpression);
}
