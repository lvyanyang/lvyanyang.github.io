# 接口文档

接口文档采用swagger规范，界面采用swagger-bootstrap-ui (opens new window)组件来实现在线查询、导出、测试。

项目接口文档访问地址：http://localhost:54321/doc.html (opens new window)

TIP

localhost:54321 表示服务器IP地址和端口，要替换为实际的IP地址和端口。

# 配置参数

#是否启用Api接口文档功能
xci.swagger.enabled=true
#是否启用接口文档权限控制
xci.swagger.auth=true
#接口文档访问账号
xci.swagger.user-name=xci
#接口文档访问密码
xci.swagger.password=xci

#接口文档标题
xci.swagger.title=西交投行业系统开发平台
#接口文档详细说明
xci.swagger.description=交通行业管理系统快速开发平台
#接口文档版本
xci.swagger.version=v1.0

#联系人姓名
xci.swagger.contact-name=
#联系人电子邮件
xci.swagger.contact-email=
#联系网址
xci.swagger.contact-url=

#许可证信息
xci.swagger.license=
#许可证网址
xci.swagger.license-url=
#服务条款网址
xci.swagger.terms-of-service-url=

# 添加新文档组

新增加一个文档组，只需要在项目 @Configuration 类中定义一个文档Bean，代码如下：

@Resource 
private CoreService coreService;

@Bean
public Docket createGroupApiDocket() {
    return coreService.buildApiDocket("组名", CoreApiController.class.getPackage().getName());
}

# 文档注解用法

# 1. @Api(类注解)

@RestController
@Api(tags = "用户接口")
public class UserApiControlle {}

- tags 接口标题。
- hidden 是否隐藏此接口，默认false。

# 2. @ApiOperation(方法注解)

@ApiOperation(value = "新建用户")
public BoolMessage insert(@RequestBody BasicDeviceCost entity);

- value 方法描述
- hidden 是否隐藏此方法，默认false。

# 3. @ApiImplicitParam(参数注解)

TIP

如果参数是对象类型，会使用对象注解，此处并不需要添加描述注解。

单个参数 
@ApiOperation(value = "删除用户")
@ApiImplicitParam(name = "ids", value = "主键字符串", required = true)
public BoolMessage delete(String ids)；

- name 参数名称
- value 参数描述
- required 是否必填项，默认false。
- allowableValues 允许填写的枚举值，多个值直接用逗号隔开。

# 4. @ApiModel(对象注解)

# 5. @ApiModelProperty(对象属性注解)

@ApiModelProperty(value = "用户名称", required = true)
private String name;

- value 表示字段描述。
- required 表示字段是必填项，默认false。
- hidden 是否隐藏此字段，默认false。
- position 设置字段顺序，默认值0，从小到大排序。
