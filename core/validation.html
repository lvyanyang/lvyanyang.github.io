# 数据校验

数据校验采用遵循JSR303–Bean Validation规范，为数据源验证定义了相应的元数据模型和API。

Hibernate Validator 是 Bean Validation 的参考实现。

  • 通过在实体字段上或者方法参数上添加各种验证注解规则
  • 通过统一的拦截验证对象是否符合规则
  • 也可以手动调用方法来验证对象是否符合规则

# 使用方法

  1. 所有继承了com.github.lvyanyang.core.BaseService类的方法或者标记org.springframework.validation.annotation.Validated注解的方法都会在方法调用时自动校验；
  2. 在实体对象字段上或者方法参数上标记验证规则注解；

注意事项

  • 如果验证对象类型的不同分组，请在对象参数前指定验证组@org.springframework.validation.annotation.Validated；
  • 如要要实现级联验证，请在相应的字段上加@Valid注解；
  • 为了尽可能逻辑复用，建议在service层使用；
  • 数据校验是使用Aop拦截来实现的，所以调用对象必须是代理对象才能生效。
  • 代理对象调用Service中的方法时会自动验证方法中的所有参数，如果有任何一个参数没有通过验证，抛出ValidException异常；
  • 校验错误消息格式化，由应用程序对象 formatValidateErrorMessage 方法来控制，默认输出格式：[字段名]错误消息。

分组验证

在设置注解规则时，可以在 javax.validation.groups.Default.class组表示默认组，是指在注解标记时，未指定分组的就是默认组； 如果只验证默认组，则默认组可以省略；
如果要同时验证默认组和其他分组，则默认组不能省略；

可以通过@GroupSequence指定分组验证顺序； @GroupSequence({First.class, Second.class, User.class})

# 消息中使用EL表达式

我们知道约束的失败消息message里是可以使用{}占位符来动态取值的，默认情况下能够取到约束注解里的所有属性值。

@Target({ElementType.FIELD, ElementType.METHOD, ElementType.PARAMETER, ElementType.ANNOTATION_TYPE})
@Retention(RetentionPolicy.RUNTIME)
@Documented
@NotNull(message = "{user.name.null}")
@Length(min = 5, max = 20, message = "{user.name.length.illegal}")
@Pattern(regexp = "[a-zA-Z]{5,20}", message = "{user.name.length.illegal}")
@Constraint(validatedBy = { })
// 组合注解一般建议标注上
@ReportAsSingleViolation 
public @interface Composition {
    String message() default "";
    Class<?>[] groups() default { };
    Class<? extends Payload>[] payload() default { };

/**
 * 性别约束验证注解

@Target({ElementType.FIELD, ElementType.METHOD, ElementType.PARAMETER, ElementType.ANNOTATION_TYPE})
@Retention(RetentionPolicy.RUNTIME)
@Documented
//指定验证器
@Constraint(validatedBy = SexConstraintValidator.class)
public @interface Sex {
    // 三个必备的基本属性

    /**
     * 默认错误消息
     */

    /**
     * 验证组
     */
    Class<?>[] groups() default {};

    /**
     * 负载
     */

    // 下面自定义属性

    //指定多个时使用
    @Target({ElementType.FIELD, ElementType.METHOD, ElementType.PARAMETER, ElementType.ANNOTATION_TYPE})
    @Retention(RetentionPolicy.RUNTIME)
    @Documented
    @interface List {
        Sex[] value();
    }
}

  1. 定义约束验证器(实现接口ConstraintValidator)
查看代码 
public class CheckPasswordValidator implements ConstraintValidator<CheckPassword, User> {

    @Override
    public void initialize(CheckPassword constraintAnnotation) {
        //初始化，得到注解属性数据
    }

    @Override
    public boolean isValid(User user, ConstraintValidatorContext context) {
        if(user == null) {
            return true;
        }

        HibernateConstraintValidatorContext hibernateContext = context.unwrap(HibernateConstraintValidatorContext.class);
        
        //请输入密码
        if(!StringUtils.hasText(user.getPassword())) {
            hibernateContext.addMessageParameter("msg", "请输入密码");
            return false;
        }

        //请输入确认密码
        if(!StringUtils.hasText(user.getConfirmation())) {
            hibernateContext.addMessageParameter("msg", "请输入确认密码");
            return false;
        }

        //两次密码输入不一致
        if (!user.getPassword().trim().equals(user.getConfirmation().trim())) {
            hibernateContext.addMessageParameter("msg", "两次密码输入不一致");
            return false;
        }
        return true;
    }
}

# 内置的验证规则

注解 说明
@Null 要求元素必须为null(所有对象，不能用于基本类型)
@NotNull 要求元素不能为null(所有对象，不能用于基本类型)
@Size 要求元素必须在指定的范围内(字符串(length)，Collection(size)，Map(size), Array(length))
@NotBlank 要求元素不能为空，要求字符串去掉不可见字符后长度大于0(字符串)
@Min(value) 要求元素必须是一个long，其值必须大于等于指定的最小值(数字)
@Max(value) 要求元素必须是一个long，其值必须小于等于指定的最大值(数字)
@Pattern 要求元素必须符合指定的正则表达式(字符串)
@AssertTrue 要求必须为true(所有对象，不能用于基本类型)
@AssertFalse 要求元素必须为false(所有对象，不能用于基本类型)
@DecimalMin(value) 要求元素必须是一个数字，其值必须大于等于指定的最小值(数字)
@DecimalMax(value) 要求元素必须是一个数字，其值必须小于等于指定的最大值(数字)
@Negative 要求元素必须是一个负数不含0(数字)
@NegativeOrZero 要求元素必须是负数或0(数字)
@Positive 要求元素必须是一个正数不含0(数字)
@PositiveOrZero 要求元素必须是一个正数或0(数字)
@Digits 要求元素必须是一个数字，小数点前后位数校验(integer:小数点前几位 fraction:小数点后几位)(数字)
@Past 要求元素必须是一个过去的日期(日期)
@PastOrPresent 要求元素必须是一个过去或当前的日期(日期)
@Future 要求元素必须是一个将来的日期(日期)
@FutureOrPresent 要求元素必须是一个将来或当前的日期(日期)

Hibernate Validator 附加的 constraint

注解 说明
@Email 要求元素必须是一个有效的邮箱地址(字符串)
@Length 要求元素的长度必须在指定的范围内(字符串)
@Range 要求元素必须在指定的范围内(数字)
@NotEmpty 要求元素不能为空(字符串(length)，Collection(size)，Map(size), Array(length))
@ScriptAssert 脚本验证器

# 注解规则提醒

WARNING

指定注解验证规则时，一定要注意注解适用的数据类型，如果类型不匹配将发生不可知异常；

比如：@NotBlank 注解用来验证字符串是否为空，如果将标记在非字符串类型的字段上将导致异常
