# 操作日志

实现业务操作日志保存。

# 使用方法

  1. 在service或者controller层方法上添加注解@OperateLog，设置msg日志消息。
  2. 配置文件xci.core.enabled-operate-log=true可以控制是否启用此功能。

注意事项

  • 为了尽可能逻辑复用，建议在service层使用
  • 操作日志是使用Aop拦截来实现的，所以调用对象必须是代理对象才能生效。
  • 添加注解@OperateLog后，如果方法执行发生了未处理的异常，拦截器会保存为一条错误的操作日志，并抛出OperateLogException异常。
  • 操作日志数据处理，由应用程序对象 processOperateLog(OperateLogModel logModel)方法来控制。
    在xci-framework-core模块中操作日志输出到控制台；
    在xci-framework-sys模块中操作日志将保存到数据库中；
    如果需要自定义数据处理逻辑，请在IApplication子类中重写此processOperateLog方法；

# 注解参数

@OperateLog注解配置参数 
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@Inherited
@Documented
public @interface OperateLog {
    /**
     * 日志消息
     */
    String msg() default Const.Empty;

    /**
     * 所属模块
     */
    String module() default Const.Empty;

    /**
     * 是否记录方法参数，如果方法参数过大，记录会影响性能，请设置为false
     */
    boolean saveParam() default true;

    /**
     * 是否记录方法返回值，如果返回值过大(比如返回列表对象)，记录会影响性能，请设置为false
     */
    boolean saveResult() default true;
}

# 手动调用

@Resource private CoreService coreService;

1.创建操作日志model
OperateLogModel.OperateLogModelBuilder builder = OperateLogModel.builder();
2.设置model属性
builder.module(logAnno.module()).status(status).msg(logAnno.msg()).costTime(usedTime).executeMethod(XCI.format("{}.{}", className, methodName));
3.处理/保存操作日志数据
coreService.processOperateLog(builder.build());

# 例子

@OperateLog(msg = "新建序列", module = SysConst.Seq.Name)
@Transactional(rollbackFor = Exception.class)
public BoolResult insert(SysSeq entity) {
    return save(entity, true);
}

@OperateLog(msg = "修改序列", module = SysConst.Seq.Name)
@Transactional(rollbackFor = Exception.class)
public BoolResult update(SysSeq entity) {
    return save(entity, false);
}
