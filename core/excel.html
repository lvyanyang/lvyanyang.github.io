# 导入导出

导入导出功能采用了EasyPoi (opens new window)开源组件实现。

使用方法：

  • 先定义实体，在实体上设置注解，用来描述导出信息；
  • 调用Api函数，传入需要导出的数据，实现导出；

实现方式是在实体类上标记注解，主要注解如下：

  • @Excel 作用到字段上,是对Excel列的描述
  • @ExcelCollection表示一个集合,主要针对一对多的导出,比如一个订单对应多个商品,商品字段就可以用集合表示
  • @ExcelEntity 表示一个继续深入导出的实体,但他没有太多的实际意义,只是告诉系统这个对象里面同样有导出的字段
  • @ExcelIgnore 和名字一样表示这个字段被忽略跳过这个导导出
  • @ExcelTarget 这个是作用于最外层的对象,描述这个对象的id,以便支持一个对象可以针对不同导出做出不同处理

# 标记实体

查看代码 
@Data
public class BasicServiceCost {
    @ExcelIgnore
    @Excel(name = "服务收费标准主键")
    private String id;
  
    @Excel(name = "收费名称", width = 20d)
    private String name;
  
    @Excel(name = "操作时间", width = 20d, exportFormat = "yyyy-MM-dd HH:mm:ss")
    @ApiModelProperty(value = "操作时间", position = 98)
    private Date operateDateTime;

    @Excel(name = "状态", replace = {"启用_true", "禁用_false"})
    private Boolean status;
}

# 导出Api

一次性读取数据并导出，适用于导出记录小于5-10万数据量的导出 
@ResponseBody
@GetMapping("/export")
public void export(SysHistoryLogFilter filter) {
    //----------------------导出Excel----------------------//

    String title = "系统历史日志";                                    //Excel标题
    String sheetName = "系统历史日志";                                //Sheet名称
    ExportParams exportParams = new ExportParams(title, sheetName); //定义Excel导出参数
    
    //导出Excel并写入到http输出流
    XCI.exportExcelToResponse(logService.selectList(filter), SysHistoryLog.class, exportParams, null);
    
    //导出Excel并写入到文件输出流
    XCI.exportExcelToFile(new File("/系统历史日志.xlsx"), logService.selectList(filter), SysHistoryLog.class, exportParams, null);

    //----------------------导出Csv----------------------//

    CsvExportParams csvExportParams = new CsvExportParams();        //定义Csv导出参数
    csvExportParams.setTextMark(Const.Empty);                       //设置记录不加引号

    //导出Csv并写入到http输出流
    XCI.exportCsvToResponse(title + ".csv", logService.selectList(filter), SysHistoryLog.class, csvExportParams, null);
    
    //导出Csv并写入到文件输出流
    XCI.exportCsvToFile(new File("/系统历史日志.xlsx"), logService.selectList(filter), SysHistoryLog.class, csvExportParams, null);
}

@ResponseBody
@GetMapping("/export")
public void export(SysHistoryLogFilter filter) {
    //设置每页写入数量
    filter.setPageSize(1000);

    //----------------------导出Excel----------------------//

    String title = "系统历史日志";                                    //Excel标题
    String sheetName = "系统历史日志";                                //Sheet名称
    ExportParams exportParams = new ExportParams(title, sheetName); //定义Excel导出参数
    
    //分页导出Excel并写入到http输出流
    XCI.exportExcelToResponse(SysHistoryLog.class, exportParams, null,
            pageIndex -> {
                filter.setPageIndex(pageIndex);
                return logService.selectPageList(filter);
            });
    
    //导出Excel并写入到文件输出流
    XCI.exportExcelToFile(new File("/系统历史日志.xlsx"), SysHistoryLog.class, exportParams, null,
            pageIndex -> {
                filter.setPageIndex(pageIndex);
                return logService.selectPageList(filter);
            });
    
    //----------------------导出CSV----------------------//

    CsvExportParams csvExportParams = new CsvExportParams();        //定义Csv导出参数
    csvExportParams.setTextMark(Const.Empty);                       //设置记录不加引号

    //分页导出Csv并写入到http输出流
    XCI.exportCsvToResponse(title + ".csv", SysHistoryLog.class, csvExportParams, null,
            pageIndex -> {
                filter.setPageIndex(pageIndex);
                return logService.selectPageList(filter);
            });

    //分页导出Csv并写入到文件输出流
    XCI.exportCsvToFile(new File("/系统历史日志.xlsx"), SysHistoryLog.class, csvExportParams, null,
            pageIndex -> {
                filter.setPageIndex(pageIndex);
                return logService.selectPageList(filter);
            });
}

# EasyPoiUtil

查看 
1.正规excel导出 (格式简单,数据量可以,5W以内吧)
注解方式:  ExcelExportUtil.exportExcel(ExportParams entity, Class<?> pojoClass,Collection<?> dataSet) 

2.不定多少列,但是格式依然简单数据库不大
自定义方式: ExcelExportUtil.exportExcel(ExportParams entity, List<ExcelExportEntity> entityList,Collection<?> dataSet)

3.数据量大超过5W,还在100W以内
注解方式 ExcelExportUtil.exportBigExcel(ExportParams entity, Class<?> pojoClass,IExcelExportServer server, Object queryParams)
自定义方式: ExcelExportUtil.exportBigExcel(ExportParams entity, List<ExcelExportEntity> excelParams,IExcelExportServer server, Object queryParams)

4.样式复杂,数据量尽量别大
模板导出 ExcelExportUtil.exportExcel(TemplateExportParams params, Map<String, Object> map)

5.一次导出多个风格不一致的sheet
模板导出 ExcelExportUtil.exportExcel(Map<Integer, Map<String, Object>> map,TemplateExportParams params) 

6.一个模板但是要导出非常多份
模板导出 ExcelExportUtil.exportExcelClone(Map<Integer, List<Map<String, Object>>> map,TemplateExportParams params)

7.模板无法满足你的自定义,试试html
自己构造html,然后我给你转成excel  ExcelXorHtmlUtil.htmlToExcel(String html, ExcelType type)

8.数据量过百万级了.放弃excel吧,csv导出
注解方式: CsvExportUtil.exportCsv(CsvExportParams params, Class<?> pojoClass, OutputStream outputStream)
自定义方式: CsvExportUtil.exportCsv(CsvExportParams params, List<ExcelExportEntity> entityList, OutputStream outputStream)

9.word导出
模板导出: WordExportUtil.exportWord07(String url, Map<String, Object> map)

# @Excel

这个是必须使用的注解,如果需求简单只使用这一个注解也是可以的,涵盖了常用的Excel需求,需要大家熟悉这个功能。

属性 类型 默认值 功能
name String null 列名,支持name_id
needMerge boolean fasle 是否需要纵向合并单元格(用于含有list的实体中,单个的单元格,合并list创建的多个row)
orderNum String "0" 列的排序,支持name_id
replace String[] {} 值得替换 导出是{a_id,b_id} 导入反过来
savePath String "upload" 导入文件保存路径,如果是图片可以填写,默认是upload/className/ IconEntity这个类对应的就是upload/Icon/
type int 1 导出类型 1 是文本 2 是图片,3 是函数,10 是数字 默认是文本
width double 10 列宽，单位为字符，一个汉字是2个字符，默认是10。
isStatistics boolean fasle 自动统计数据,在追加一行统计,把所有数据都和输出[这个处理会吞没异常,请注意这一点]
isHyperlink boolean false 超链接,如果是需要实现接口返回对象
isImportField boolean true 校验字段,看看这个字段是不是导入的Excel中有,如果没有说明是错误的Excel,读取失败,支持name_id
exportFormat String "" 导出的时间格式,以这个是否为空来判断是否需要格式化日期
importFormat String "" 导入的时间格式,以这个是否为空来判断是否需要格式化日期
format String "" 时间格式,相当于同时设置了exportFormat 和 importFormat
databaseFormat String "yyyyMMddHHmmss" 导出时间设置,如果字段是Date类型则不需要设置 数据库如果是string 类型,这个需要设置这个数据库格式,用以转换时间格式输出
numFormat String "" 数字格式化,参数是Pattern,使用的对象是DecimalFormat
imageType int 1 导出类型 1 从file读取 2 是从数据库中读取 默认是文件 同样导入也是一样的
suffix String "" 文字后缀,如% 90 变成90%
isWrap boolean true 是否换行 即支持\n
mergeRely int[] {} 合并单元格依赖关系,比如第二列合并是基于第一列 则{0}就可以了
mergeVertical boolean fasle 纵向合并内容相同的单元格(垂直方向内容相同的单元格会合并)
fixedIndex int -1 对应excel的列,忽略名字
isColumnHidden boolean false 导出隐藏列
groupName String "" 导出时，表头双行显示,聚合

# @ExcelCollection

一对多的集合注解,用以标记集合是否被数据以及集合的整体排序

属性 类型 默认值 功能

name String null 定义集合列名,支持nanm_id
orderNum int 0 排序,支持name_id
type Class<?> ArrayList.class 导入时创建对象使用
