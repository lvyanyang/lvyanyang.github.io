# 分页查询

实现对数据表的分页查询，支持单表、多表连接查询。

# 使用方法

  1. 在 dao 层查询分页方法上添加注解 @Paging，返回类型为 java.util.List 或者 com.github.pagehelper.Page。
  2. dao 层分页方法必须要包含一个继承自 BasePageFilter 的对象，用来传递分页参数。
  3. MyBatis Xml 文件中写标准 select 查询语句。
分页请求参数 
{
	"pageIndex": 1,// 页码,默认:1
	"pageSize": 10,// 每页记录数,默认:10
	"sortName": "",// 排序字段名，启用多字段排序后，多个排序字段名之间用逗号分隔
	"sortDir": ""  // 排序方式，启用多字段排序后，多个排序方式之间用逗号分隔，可选值[asc|desc]
}

# 注解参数

@Paging注解配置参数 
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@Inherited
public @interface Paging {
    /**
     * 指定允许排序的字段名集合，如果不指定则排序功能无法使用
     */
    String[] allowSortNames() default {};

    /**
     * 手动指定排序字段名转换，格式：[原始排序字段名:转换后排序字段名]
     */
    String[] sortNameMaps() default {};

    /**
     * 排序字段名转换，可选值[Default-不转换,ToUnderlineCase-驼峰转下划线,ToCamelCase-下划线转驼峰]
     * 如果 sortNameMaps 属性中指定了排序字段名转换，则此选项将无效，sortNameMaps 中未指定的排序字段名将按此规则转换
     */
    SortStringConvertType sortNameConvertType() default SortStringConvertType.Default;

    /**
     * 是否启用多字段排序功能
     */
    boolean enableMultipleSort() default false;

    /**
     * 设置为true且pageSize=0时返回全部结果，false且pageSize=0时抛出异常
     */
    boolean pageSizeZero() default false;

    /**
     * 每页最大记录数，默认-1表示不指定，由全局配置参数 xci.core.max-page-size 进行控制
     */
    int maxPageSize() default -1;

    /**
     * 未指定页码时的默认值
     */
    int defaultPageIndex() default 1;

    /**
     * 未指定每页记录数时的默认值
     */
    int defaultPageSize() default 10;

    /**
     * 未指定排序字段名时的默认值
     */
    String defaultSortName() default Const.Empty;

    /**
     * 未指定排序方式时的默认值，可选值[asc|desc]
     */
    String defaultSortBy() default Const.Empty;
}

    select id,code,name,start_with,current_value,increment_by,remark from sys_seq

手动指定count查询SQL 
<!--
分页插件会优先通过当前查询的 msId + countSuffix 查找手写的分页查询。
如果存在就使用手写的 count 查询，如果不存在，仍然使用之前的方式自动创建 count 查询。
-->

<select id="selectLeftjoin" resultType="com.github.pagehelper.model.User">
    select a.id,b.name,a.py from user a
    left join user b on a.id = b.id
    order by a.id
</select>
<select id="selectLeftjoin_COUNT" resultType="Long">
    select count(distinct a.id) from user a
    left join user b on a.id = b.id
</select>

<!--
上面的 countSuffix 使用的默认值 _COUNT，分页插件会自动获取到 selectLeftjoin_COUNT 查询，这个查询需要自己保证结果数正确。
返回值的类型必须是resultType="Long"，入参使用的和 selectLeftjoin 查询相同的参数，所以在 SQL 中要按照 selectLeftjoin 的入参来使用。
-->
