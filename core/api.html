# 接口规范

# 请求方法

无特殊说明，默认全部使用POST方法请求

# 请求参数

无特殊说明，默认全部使用Json格式发送

  • 接收参数用 @RequestBody 注解标注
  • 返回参数用 @ResponseBody 注解或者 @RestController 标注

提示

文件上传和下载除外

# 公共Header请求参数

名称 值 说明
X-Requested-Type APIHttpRequest 接口请求标识(必填)

如果加载了 xci-framework-sys 模块并且前缀以api开头的接口，还需要添加一下参数

名称 值 说明
appId 配置值 调用方标识
token 用户token 登录成功后返回

# 响应格式

{
  "success": false,//请求状态，true表示请求调用成功，false表示请求调用失败。
  "code": 0,       //请求状态码，大于等于0请求调用成功，小于0表示请求调用失败。
  "msg": "",       //请求消息，多用来返回错误消息。
  "data": {}       //请求相关的业务数据，data可以是数字、字符串、日期、对象、集合以及可序列化的所有对象。
}
