# 单个Json参数

目前按照规范要求，接口所有的参数都要使用json格式进行传递，那么服务端要接收参数，必须要用类来接收。

比如，按照id获取某条记录，那么需要服务端定义一个类，并添加id属性来接收，代码如下：

接口

@PostMapping(value = "/selectById")
public BoolResult<SysSeq> selectById(@RequestBody IdBody body) {
    return BoolResult.success(seqService.selectById(body.getId()));
}

@Getter

@ApiModel(description = "Long主键Body参数")
public class IdBody {
    @ApiModelProperty(value = "主键", required = true)

这样会导致产生大量的临时类，写起来比较麻烦。

为了解决这个问题，框架增加了RequestJson注解，直接标记在参数前面，而无需定义类。建议超过3个参数就要建立参数接收类，否则不易维护。

@ApiOperation(value = "查询数据")
@ApiImplicitParams({
    @ApiImplicitParam(name = "id", value = "主键"),
    @ApiImplicitParam(name = "name", value = "名称")
})
@PostMapping(value = "/selectById")
public BoolResult<SysSeq> selectById(@RequestJson Long id,@RequestJson String name) {
    return BoolResult.success(seqService.selectById(id));
}
