# 应用程序接口

应用程序扩展接口实现内部功能扩展，主要扩展点如下：

  • 允许自定义处理Api全局异常（主要针对Api接口请求的异常）
  • 允许自定义处理Ajax全局异常（主要针对Ajax接口请求的异常）
  • 允许自定义处理页面全局异常（主要针对Page请求的异常）
  • 允许自定义如何获取当前操作用户
  • 允许自定义数据校验错误消息如何显示
  • 允许自定义处理操作日志对象
  • 允许自定义处理异常错误对象

TIP

  • 每个应用程序只能定义一个 IApplication
  • 系统核心模块默认的实现基类是 BaseApplication
  • 系统权限模块的实现基类是 SysApplication

使用方法：

启动类可以继承自 BaseApplication，如果加载了 xci-framework-sys模块，需要继承自 SysApplication。

@PropertySource("classpath:xci.properties")
@SpringBootApplication
public class TestApplication extends BaseApplication {
    public static void main(String[] args) {
        SpringApplication.run(TestApplication.class, args);
    }
}

# IApplication

查看 
/**
 * 应用程序扩展接口
 */
public interface IApplication {
    /**
     * 处理Api全局异常
     * http请求Header中存在[X-Requested-Type=APIHttpRequest]键值对则识别为api请求
     * @param e 异常对象
     */
    BoolResult handleApiGlobalException(Exception e);

    /**
     * 处理Ajax全局异常
     * http请求Header中存在[X-Requested-With=XMLHttpRequest]键值对则识别为ajax请求
     * @param e 异常对象
     */
    BoolResult handleAjaxGlobalException(Exception e);

    /**
     * 处理Page全局异常
     * 除了api请求和ajax请求之外的请求均为页面请求
     * @param e 异常对象
     */
    ModelAndView handlePageGlobalException(Exception e);

    /**
     * 格式化数据校验错误消息
     * @param errorMessageList 错误消息列表
     */
    String formatValidateErrorMessage(List<ValidateErrorMessage> errorMessageList);

    /**
     * 获取当前操作用户对象
     */
    IUser getCurrentUser();

    /**
     * 处理操作日志对象
     * @param logModel 日志信息
     */
    void processOperateLog(OperateLogModel logModel);

    /**
     * 处理异常错误对象
     * @param e 异常对象
     */
    void processErrorLog(Exception e);
}

# BaseApplication

  • handleApiGlobalException 按照接口开发调用规范返回错误信息(读取配置文件)并调用processErrorLog记录日志
  • handleAjaxGlobalException 按照接口开发调用规范返回错误信息(读取配置文件)并调用processErrorLog记录日志
  • handlePageGlobalException 把错误信息转发到指定的错误页(读取配置文件)并调用processErrorLog记录日志

默认情况，以下异常属于已知的异常，已进行预处理，只会把错误返回给客户端，不会记录到错误日志中

MaxUploadSizeExceededException
NotFoundDataException
org.apache.ibatis.javassist.NotFoundException
HttpRequestMethodNotSupportedException
HttpMessageNotReadableException
BusinessException
OperateLogException
ValidException

  • formatValidateErrorMessage 格式化数据校验错误消息，默认格式：[字段名]错误消息
  • getCurrentUser 返回null，由应用程序来实现
  • processOperateLog 只是输出在控制台
  • processErrorLog 只是输出在控制台
查看 
@Slf4j
public class BaseApplication implements IApplication {
    @Resource private CoreProperties coreProperties;
    @Resource private CoreService coreService;

    @Override
    public BoolResult handleApiGlobalException(Exception e) {
        return coreService.coreGlobalExceptionHandler(e);
    }

    @Override
    public BoolResult handleAjaxGlobalException(Exception e) {
        return coreService.coreGlobalExceptionHandler(e);
    }

    @Override
    public ModelAndView handlePageGlobalException(Exception e) {
        var result = coreService.coreGlobalExceptionHandler(e);
        String errorUrl = coreProperties.getCore().getError500Url();
        return XCI.getExceptionView(e, errorUrl, result.getMsg());
    }

    @Override
    public String formatValidateErrorMessage(List<ValidateErrorMessage> errorMessageList) {
        return XCI.formatValidateErrorMessage(errorMessageList);
    }

    @Override
    public IUser getCurrentUser() {
        return null;
    }

    @Override
    public void processOperateLog(OperateLogModel logModel) {
        log.warn("保存操作日志:{}", XCI.toJsonString(logModel));
    }

    @Override
    public void processErrorLog(Exception e) {
        log.error("保存错误日志:", e);
    }
}
