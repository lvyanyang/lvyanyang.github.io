# Exist转换

在项目开发过程中经常需要按照where条件检测记录是否存在，功能看似简单，不同的数据库语法差异较大，看下面的代码：

--Oracle语法
select case when exists(select 1 from user where code = #{code} and id != #{excludeId}) then 1 else 0 end from dual

--MySQL/SQLServer/SQLite语法
select case when exists(select 1 from user where code = #{code} and id != #{excludeId}) then 1 else 0 end

当然有的解决方案是采用count来实现，当记录数量较多时会存在重大的性能损失。

为了简化这个问题，框架自定义了MyBatis插件 ExistxInterceptor，会根据当前数据源的类型，自动转换查询的SQL。

# 转换原理

  1. MyBatis插件拦截所有 select 查询语句，检测方法名是否以existx开通，如果是则执行替换逻辑，否则跳过。
  2. 先获取原始sql,提取表名以及where后面的语句(不带where)。
  3. 根据当前数据源，生成不同数据库的exist语句。

# 使用方法

  1. 在配置文件设置 xci.core.enabled-existx=true，来启用此功能，默认为启用状态
  2. 在 dao 层添加方法，并已 existx 开头，目前只支持单表条件，对于其他复杂sql暂时没有解析，后续支持。
  3. MyBatis Xml 文件中写 select count(1) from user 语句即可，插件根据当前数据源自动转换，如果不支持将采用select count(1)来实现，目前支持的数据库：oracle、sqlserver、mysql、sqlite。

# 例子

  1. Dao层
/**
 * 检查参数编码是否存在
 * @param code      参数编码
 * @param excludeId 排除的主键，如果为null则不指定排除的主键
 * @return 如果存在返回true
 */
boolean existxByCode(@Param("code") String code, @Param("excludeId") Long excludeId);

  1. Mybatis XML
<!--region 检查参数编码是否存在-->
<select id="existxByCode" resultType="boolean">
    select count(1) from sys_param
    where code = #{code}
    <if test="excludeId != null">and id != #{excludeId}</if>
</select>
<!--endregion-->
