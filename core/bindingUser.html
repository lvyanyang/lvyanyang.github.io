# 绑定用户

在日常开发中，经常需要记录当前操作人员信息及操作时间等信息，在框架中也进行了简单封装。

# 使用方法

在 Controller 或者 Service 方法上使用 BindingOperateUser或BindingCreateUser或BindingUpdateUser 注解

具体规则如下：

  • BindingOperateUser 需要参数对象继承 BaseOperateUserEntity 对象，绑定 operateUserId、operateUserName、operateDateTime 字段。
  • BindingCreateUser 需要参数对象继承 BaseUserEntity 对象，绑定 createUserId、createUserName、createDateTime 字段。
  • BindingUpdateUser 需要参数对象继承 BaseUserEntity 对象，绑定 updateUserId、updateUserName、updateDateTime 字段。

TIP

通过Aop拦截方法调用，实现对参数绑定当前操作用户信息和时间信息，只能绑定到对象类型；

支持绑定多个参数对象；

当前操作用户信息来源，请参考 IApplication 章节，getCurrentUser 方法的定义。

# 手动调用

  1. 从Spring容器获取 CoreService 实例对象：@Resource private CoreService coreService。
  2. 调用 coreService.getCurrentUser() 方法返回IUser对象，返回用户主键、账号、姓名字段。

TIP

如果加载了 xci-framework-sys 模块，可以使用@Resource private SysService sysService、sysService.getCurrentUser() 返回SysUser对象返回用户完整信息。

# 自定义绑定注解

如果内置注解 BindingOperateUser、BindingCreateUser、BindingUpdateUser 无法满足需求，可以自定义注解实现自定义逻辑，好处是复用性较高。

  1. 新建 ApplyUser 注解拦截器 BindingApplyUserAspect

/**
 * 绑定当前申请用户字段信息拦截器
 */
@Aspect
@Component
public class BindingApplyUserAspect {
    @Resource private CoreService coreService;

    @Before("@annotation(ApplyUser)")
    public void doProcess(JoinPoint joinPoint) {
        Object[] args = joinPoint.getArgs();
        if (args.length == 0) return;
        for (Object bean : args) {
            if (bean == null) continue;
            if (bean.getClass().isPrimitive()) continue;
            coreService.setCustomUserInfo(bean,"applyUserId","applyUserName","applyDateTime");
        }
    }
}
